# CS230

The client for this project was The Gaming Room, which wanted to expand its Android game, Draw It or Lose It, into a web-based application that could run on multiple operating systems. The software needed to support multiple teams and players, ensure unique game and team names, and maintain a single game service using the Singleton design pattern. I believe I did well in organizing the software design document and supplying clear recommendations for system architecture, platform compatibility, memory management, and software requirements. Completing the design document before coding helped me better understand the application's structure and made the implementation process more organized and efficient.

If I could revise one part of my work, I would expand the evaluation section by providing more detailed comparisons of security, scalability, and deployment options. Compare each option more directly so the evaluation shows clear trade-offs and stronger distinctions. I interpreted the client's needs by translating the business requirements into technical design decisions to create a reliable, user-friendly application. Considering the user's needs is essential because software should solve the client's problems while providing a positive experience. In future projects, I will continue to use structured planning, UML diagrams, design patterns, and careful analysis of requirements to develop software that is maintainable, scalable, and that meets both user and business goals.
